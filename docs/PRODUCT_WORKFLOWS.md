# Product Workflows

## 1. Public entry workflow

### Goal
Get a visitor from curiosity to trust to action.

### Standard path
1. visitor lands on homepage
2. understands the core promise
3. reviews How It Works
4. reviews plans / trust content
5. chooses one of the main actions:
   - subscribe / buy
   - start intake
   - book consultation

## 2. Pre-purchase consultation workflow

### Goal
Qualify the user and recommend the right plan.

### Working flow
1. user begins intake or consultation questionnaire
2. platform gathers IP type, risk, goals, and urgency
3. platform scores or categorizes user readiness
4. platform recommends plan / next step
5. user proceeds to purchase or booking

### Intended outputs
- plan recommendation
- readiness score or summary
- clearer onboarding path

## 3. Purchase / subscription workflow

### Goal
Move the user into the correct recurring or one-time product path.

### Working flow
1. user selects plan
2. billing is completed
3. user receives confirmation
4. platform triggers onboarding and access logic
5. user is routed into dashboard / vault / next action

## 4. Timestamp workflow

### Goal
Turn uploaded IP material into evidence-ready proof-of-existence flow.

### Working flow
1. user uploads file or asset
2. system computes file hash
3. system records evidence metadata
4. system anchors or queues anchoring process
5. system generates certificate / status output
6. user can access evidence artifact later

### Important posture
This workflow is evidence-first and proof-of-existence oriented.
It is not the same as granting statutory IP rights.

## 5. Certificate workflow

### Goal
Return a professional, user-readable evidence artifact.

### Working flow
1. completed timestamp generates evidence payload
2. certificate template is populated
3. file details, hash, chain / transaction references, and verification details are inserted
4. certificate is stored and made downloadable
5. user can later export supporting evidence package

## 6. Post-purchase onboarding workflow

### Goal
Get the user from purchase to first successful protected asset as quickly as possible.

### Working flow
1. platform welcomes user into vault / dashboard
2. user sees guided first action
3. platform encourages first upload and first timestamp
4. certificate is generated
5. dashboard shows next recommended steps

### Intended feeling
The user should feel that the platform is:
- serious
- clear
- safe
- premium
- actually useful on day one

## 7. Vault / dashboard workflow

### Goal
Give the user a stable place to manage evidence, assets, and next steps.

### Core dashboard functions
- asset list
- timestamp status
- certificate access
- account / subscription visibility
- next-action prompts
- evidence export

## 8. Partner law-firm workflow

### Goal
Allow qualified user matters to move toward legal help without forcing every user into that path.

### Working flow
1. user opts into showcase or referral path
2. relevant data is filtered / anonymized if needed
3. partner firms browse or receive leads
4. interest is expressed
5. introduction or proposal handoff happens
6. commissions / partner events are tracked

## 9. Admin workflow

### Goal
Provide oversight of product, users, tiers, and partner activity.

### Admin needs
- subscription state
- workflow state
- timestamp / certificate records
- partner activity
- KPI and revenue visibility
- issue / support handling

## 10. Future enhancement workflow ideas

These are product-adjacent ideas discussed earlier and should be treated as staged enhancements, not automatic MVP requirements:

- novelty / prior-art style check after onboarding
- stronger court-evidence export packaging
- advanced partner dashboards
- deeper white-label flows
- AI-assisted consultation and scoring improvements
